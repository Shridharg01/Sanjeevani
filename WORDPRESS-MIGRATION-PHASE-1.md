## Step 1.4: SSL & Security Setup (Day 2 - 45 minutes)

### SSL Configuration

1. **Verify SSL is Active**
   - hPanel → Website Settings → SSL
   - Should show: "SSL Certificate Active" with auto-renewal enabled
   - Installation type: Auto-installed by Hostinger

2. **Force HTTPS**
   - Via WordPress:
     - Admin → Settings → General
     - WordPress Address (URL): `https://sanjeevanicosderma.in`
     - Site Address (URL): `https://sanjeevanicosderma.in`
     - Save

3. **Configure .htaccess for HTTPS Redirect**

   Via File Manager → public_html → .htaccess

   Add at the top (before WordPress rules):

   ```apache
   <IfModule mod_rewrite.c>
   RewriteEngine On
   RewriteCond %{HTTPS} off
   RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
   RewriteCond %{HTTP_HOST} ^www\.
   RewriteRule ^(.*)$ https://sanjeevanicosderma.in/$1 [L,R=301]
   </IfModule>
   ```

4. **Test SSL**
   - Visit: https://www.ssllabs.com/ssltest/
   - Enter: sanjeevanicosderma.in
   - Should receive grade A or A+

### Security Headers

Add to .htaccess (after HTTPS redirect):

```apache
<IfModule mod_headers.c>
Header set X-Content-Type-Options "nosniff"
Header set X-Frame-Options "SAMEORIGIN"
Header set X-XSS-Protection "1; mode=block"
Header set Referrer-Policy "strict-origin-when-cross-origin"
Header set Permissions-Policy "geolocation=(), microphone=(), camera=()"
</IfModule>
```

## Troubleshooting Guide

### Issue: WordPress installation fails
- **Solution:** Check PHP version in Hostinger (should be 8.2)
- **Solution:** Verify database permissions
- **Solution:** Contact Hostinger support

### Issue: Plugins not activating
- **Solution:** Check PHP memory limit (should be 512M)
- **Solution:** Deactivate all plugins, reactivate one by one
- **Solution:** Check plugin error log: /wp-content/debug.log

### Issue: LiteSpeed Cache not showing
- **Solution:** Verify LiteSpeed Cache plugin installed
- **Solution:** Visit frontend page and refresh (cache needs to build)
- **Solution:** Check browser response headers for X-LiteSpeed-Cache

### Issue: CloudFlare not caching
- **Solution:** Verify CloudFlare integration active in hPanel
- **Solution:** Check CloudFlare cache level set to "Cache Everything"
- **Solution:** Add /wp-admin and /wp-login to bypass rules

### Issue: HTTPS redirect loops
- **Solution:** Check .htaccess for duplicate HTTPS rules
- **Solution:** Verify WordPress URLs use https:// in settings
- **Solution:** Clear CloudFlare cache and browser cache
